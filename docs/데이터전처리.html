Technical Document

데이터 전처리 및 학습된 인공지능 모델

문서 분류: 내부 대외비

1. 모델 개요 및 적용 환경

모델 목적 사용자가 업로드한 CAD 도면 파일(JSON 구조 변환)을 분석하여 건축(arch), 배관(pipe), 소방(fire), 전기(elec) 중 하나의 도메인으로 자동 분류
알고리즘 CatBoostClassifier (대칭 트리 기반 그래디언트 부스팅)
적용 위치 Backend Server (services/agent_service.py) 내 DomainClassifier 래퍼 클래스로 통합 구동
저장 포맷 domain_classifier.pkl label_encoder.pkl domain_classifier_meta.json

2. 입력 데이터 명세

Feature Vector: 74차원

도면에서 추출된 기하 및 의미론적 특성은 아래 74차원의 실수(Float) 배열로 정규화되어 입력됩니다.

[A]엔티티 타입 비율 (11차원)

LINE, TEXT, HATCH, CIRCLE 등 엔티티 빈도를 전체 개수로 나눈 비율

[B]이진 존재 여부 (3차원)

도메인 특화 기하학 요소인 HATCH, ELLIPSE, SPLINE의 존재 여부 (0 or 1)

[C,D,F,G,H,K,L,M,N]구조/기하 통계 피처 (28차원)

파생 엔티티 비율합, 도면 스케일 단위(Unit) 원핫 인코딩, 레이어명 길이(평균/표준편차), 레이어명 스타일(한글/특수문자 비율), AIA 표준 접두어(A-/E-/F- 등) 비율, 색상 다양성 엔트로피, Bbox 면적 통계 등

[O]레이어 도메인 키워드 가중 점수 (8차원)

도메인별 고유 키워드(예: 소방 spr, 전기 MCC) 매칭 시 최대 2.0~1.0 가중치를 부여한 합산 점수 및 히트 비율

[P]TEXT 정규식 패턴 분석 (8차원)

MTEXT 내부 텍스트를 파싱하여 관경(Ø50), 전기스펙(220V), 건축실명(거실), 스프링클러 사양(K-80) 등 8가지 핵심 카테고리 매칭 비율 계산 (오탐 방지를 위한 부정 전방탐색 정규식 적용)

3. 출력 데이터 및 Fallback 명세

Single Domain
단일 도메인 반환

predict() 호출 시 최종 판별된 도메인 문자열 반환 (예: "elec")

Probability Distribution
확률 분포 반환

predict_proba() 호출 시 각 도메인별 신뢰도 딕셔너리 반환 ({"arch": 0.05, "elec": 0.82, "fire": 0.08, "pipe": 0.05})

Batch Processing
배치 처리

predict_batch()를 통해 복수 도면 동시 처리 지원 (numpy.stack 병렬 연산)

Safety Net
안전망 (Fallback) 로직

모델 파일 누락, 패키지 충돌, 피처 추출 중 심각한 예외 발생 시 서비스 중단을 막기 위해 기본값인 "arch"를 반환하고 WARNING 로그를 남기도록 설계되었습니다.

4. 최종 모델 성능 평가

Test Set: 131개 기준

다출처 도면 435개 데이터셋을 기준으로 한 성능 지표입니다.

도메인별 세부 성능 (CV F1 기준)

배관 (pipe) 0.8681 관경 패턴 및 재질 키워드 신호가 강해 최고 성능
전기 (elec) 0.8462 전압/전선 정규식 및 색상 다양성 신호 우수
건축 (arch) 0.7956 도면 종류 변동성이 크나 층고/실명 패턴으로 선방
소방 (fire) 0.6982 배관과 ARC/CIRCLE 기하를 강하게 공유하여 구별 난이도 높음

5. 배포 스펙 및 운영 환경

필요 라이브러리
scikit-learn==1.8.0 catboost==1.2.10 numpy>=1.26 joblib
추론 스펙
GPU가 불필요하며, CPU 환경에서 단일 샘플당 피처 추출 포함 < 10ms 이내의 고속 추론 보장

6. 향후 고도화 및 개선 방향

  • 소방 도면 한계 극복: 소방 성능(F1 0.69) 개선을 위해 텍스트 주석이 없는 도면에서도 구별할 수 있도록 '블록 삽입점 간의 간격 분포' 같은 세밀한 기하 피처 연구 및 소방 데이터 150개 이상 추가 확보.
  • 데이터 한계 돌파: 현재 N=435 규모의 데이터를 1,000개 이상으로 다출처 구축할 경우 Test Macro F1 0.85 이상 달성 예상.
  • Human-in-the-Loop 적용: predict_proba() 결과 중 최댓값이 0.6 미만인 '경계선상 도면(예: 배관/소방 혼합)'의 경우, 에이전트가 단독 결정하지 않고 사용자에게 도메인 선택 UI를 노출하여 정확도를 끌어올리는 사용자 개입 시스템 도입 예정.
