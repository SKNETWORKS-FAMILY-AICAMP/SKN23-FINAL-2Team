# Memory Text 저장 가이드

## 1. recent_chat 추가 포맷

새로운 대화 1턴은 반드시 아래 형식으로 저장합니다.

```text
User: {질문}
AI: {답변}
```

턴과 턴 사이는 아래 구분자로 나눕니다.

```text
---
```

즉 `recent_chat` 예시는 다음과 같습니다.

```text
User: 복도 폭 규정을 확인해줘
AI: 복도 폭은 1200mm 이상 기준을 우선 확인해야 합니다.

---

User: door_101도 같이 봐줘
AI: door_101 객체는 현재 검토 대상에 포함했습니다.
```

## 2. summary_text 와 recent_chat 프롬프트 주입 표준

LLM 호출 시에는 두 값을 아래 순서로 결합합니다.

```text
[누적 요약]
{summary_text}

[최근 대화 원문]
{recent_chat}
```

`summary_text` 는 장기 기억용 압축 정보이고,  
`recent_chat` 은 최근 5턴의 원문 맥락 유지용입니다.

## 3. 왜 JSONB 대신 TEXT 두 컬럼인가

이 구조는 다음 목적을 위해 사용합니다.

- JSONB 전체 상태를 매번 직렬화 역직렬화하는 비용을 줄입니다.
- LLM 에 실제로 필요한 정보만 텍스트로 바로 주입하기 쉽습니다.
- 최근 원문과 장기 요약을 분리해서 컨텍스트 길이를 안정적으로 관리할 수 있습니다.
- 요약본은 장기 기억, recent_chat 은 단기 기억 역할을 하므로 프롬프트 구성이 단순해집니다.
